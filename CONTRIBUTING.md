# Руководство по внесению вклада (Contributing)

Спасибо за интерес к этому проекту! Этот документ описывает процесс настройки окружения и внесения изменений.

## 🛠 Требования

- Go 1.21+
- Docker & Docker Compose

## 🧪 Тестирование и Линтинг

Перед отправкой Pull Request убедитесь, что все тесты проходят и код отформатирован.

## 📝 Стиль кода и Коммиты

- Мы следуем стандартным правилам форматирования Go (`go fmt`).
- Для коммитов используется соглашение [Conventional Commits](https://www.conventionalcommits.org/).
  - Пример: `feat: add redis caching layer`
  - Пример: `fix: resolve race condition in worker`

## 🔄 Pull Request Process

1. Создайте ветку от `main` (`feature/your-feature-name`).
2. Внесите изменения и напишите тесты.
3. Убедитесь, что CI пайплайн зеленый.
4. Создайте PR и опишите, что было изменено.
